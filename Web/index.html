web architecture

In this example, there will be a client using Firefox, and it will look for the URL address google.com, to be able to access the web and see in this case the web application, the client's device is doing all these processes(in the backgrounf) that we will explain below:

The first thing the browser will do is use a DNS server to find out which IP represents the google.com domain. Basically the client's machine makes a request to the DNS server, the DNS server responds with the IP corresponding to the domain that the client has searched for in their browser

Once it knows the IP it needs to access, the client machine sends a TCP request, where it basically checks if the server is available.

When the server responds positively, therefore it is available and accepts the client's requests, the client and the server agree on which protocol to use, usually HTTP or HTTPS, first of all the browser try to connect with HTTPS (because the data travels encrypted thanks to SSL certificates) and if it's not possible then uses the HTTP

Once they have decided which protocol they will use to communicate via the Web, it is the moment when the web server sends the HTML page to the client, and finally the browser is responsible for interpreting the HTML and displaying the web
